# PLANS.md - 系统开发计划

## 一、系统架构规划

### 1.1 用户角色与权限
| 角色 | 权限范围 | 主要功能 |
|------|---------|---------|
| 计划员(planner) | 全系统 | 导入发货计划、倒推计算、审批计划、查看所有班组 |
| 班组(team) | 本班组 | 排次日生产计划、提交计划、申请退回 |

### 1.2 模块划分
1. **工作台**: 今日概览、预警统计、在产订单数
2. **预警中心**: 按交期倒推，自动标红/黄/绿
3. **班组排班**: 甘特图+排班表，支持拖拽调整
4. **订单管理**: 工单导入、搜索、编辑
5. **产品管理**: 工艺路线、生产周期、物料清单(各自支持导入导出)
6. **用户管理**: 管理员创建/编辑用户

## 二、数据流设计

### 2.1 发货计划倒推流程
```
发货计划 → 倒推计算 → 生成半成品需求 → 显示在预警中心
                ↑
        生产周期 + 安全库存
```

### 2.2 排班流程
```
工单导入 → 搜索工单 → 选择工序 → 选择设备 → 填写产能/工时 → 保存排班
                              ↓
                    从工单进度自动填充
```

### 2.3 计划审批流程
```
班组创建草稿 → 提交 → 计划员审批 → 通过/拒绝
                    ↓
              申请退回 ← 班组修改 → 重新提交
```

## 三、甘特图设计

### 3.1 显示规则
- 横轴: 时间(08:00-20:00)
- 纵轴: 设备列表
- 任务条显示: 产品名称 + 工序 + 数量 + 生产人
- 颜色: P0红/P1蓝/P2灰

### 3.2 交互功能
- 左右拖拽: 调整时间(30分钟吸附，跳过休息)
- 上下拖拽: 更换设备
- 单击: 编辑弹窗
- 双击空白: 新增排班
- 悬停: 显示详细信息(跟随鼠标)

## 四、工单工序进度

### 4.1 数据格式
工单的process_progress字段格式:
```
工序名【完成数/总数】（百分比%）->工序名【完成数/总数】（百分比%）
```
示例:
```
焊接压板1（自动）【860/1500】（57%）->焊接开关座（手动）【860/1500】（57%）->半成品入库【510/1500】（34%）
```

### 4.2 使用方式
1. 选择工单后，自动解析工序进度
2. 工序下拉框只显示该工单的工序(而非全部工艺路线)
3. 工序按进度显示状态: ✓完成/57%进行中/待开始
4. 点击进度卡片可快速选择工序
5. 自动填充剩余数量

## 五、数据导入模板

### 5.1 工单模板(工单.xlsx)
| 列 | 字段 | 说明 |
|----|------|------|
| A | 工单号 | 唯一标识 |
| C | 产品编码 | 关联产品 |
| D | 状态 | 执行中/已完成等 |
| E | 工序进度 | 格式: 工序名【完成/总数】（%） |
| F | 数量 | 计划数量 |
| G | 优先级 | P0/P1/P2 |
| K | 交期 | 日期 |
| N | 已完成数量 | 实际完成数 |

### 5.2 工艺路线模板
| 列 | 字段 | 说明 |
|----|------|------|
| A | 产品编码 | 精确匹配 |
| B | 路线编码 | 唯一标识 |
| C | 路线名称 | 描述 |
| D | 工序列表 | 逗号分隔: 工序1,工序2,工序3 |
