# implement.md - 实现细节文档

## 一、核心API接口

### 1.1 认证相关
| 接口 | 方法 | 说明 |
|------|------|------|
| /api/login | POST | 登录，返回session |
| /api/logout | POST | 登出 |
| /api/me | GET | 获取当前用户信息 |

### 1.2 排班相关
| 接口 | 方法 | 说明 |
|------|------|------|
| /api/schedule | GET | 获取排班列表(team_id, date) |
| /api/schedule | POST | 创建排班 |
| /api/schedule/:id | PUT | 更新排班 |
| /api/schedule/:id | DELETE | 删除排班 |

### 1.3 工单相关
| 接口 | 方法 | 说明 |
|------|------|------|
| /api/orders | GET | 工单列表 |
| /api/search-orders?q= | GET | 搜索工单 |
| /api/product-processes?product_code= | GET | 获取产品工序 |

### 1.4 日计划相关
| 接口 | 方法 | 说明 |
|------|------|------|
| /api/daily-plan | GET | 获取/创建日计划 |
| /api/daily-plan/:id/submit | POST | 提交计划 |
| /api/daily-plan/:id/approve | POST | 审批通过 |
| /api/daily-plan/:id/return | POST | 退回 |

## 二、甘特图实现

### 2.1 时间计算
工作时间: 08:00-17:00
休息时间: 10:00-10:10, 11:45-12:45, 15:00-15:10
加班时间: 17:30-20:00 (仅当日)

### 2.2 吸附规则
- 30分钟为单位吸附
- 跳过休息时间段
- 开始时间必须在休息之后
- 结束时间必须在休息之前

### 2.3 拖拽功能
- 左边缘拖拽: 调整开始时间
- 右边缘拖拽: 调整结束时间
- 中间拖拽: 整体移动(可更换设备)

## 三、工序进度解析

### 3.1 数据格式
工单的process_progress字段格式:
工序名【完成数/总数】（百分比%）->工序名【完成数/总数】（百分比%）

### 3.2 解析逻辑
使用正则表达式解析:
- 工序名称: (.+?)
- 完成数: (\d+)
- 总数: (\d+)
- 百分比: (\d+)

### 3.3 使用流程
1. 用户搜索并选择工单
2. 解析工单的process_progress字段
3. 显示进度卡片(颜色区分完成度)
4. 工序下拉框只显示该工单的工序
5. 点击进度卡片可选择对应工序
6. 自动填充剩余数量

写Python脚本时使用@' ... '@语法并保存到文件执行。
中文字符在控制台可能显示乱码，但文件内容是正确的。
