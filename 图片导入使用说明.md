## 图片下载方案

### 方案1：Web界面下载（推荐）
1. 访问产品定义页面
2. 点击"🖼️ 下载全部图片"按钮
3. 系统自动在后台下载所有图片
4. 下载完成后刷新页面查看图片

**优点**：
- 简单易用
- 后台运行，不阻塞操作
- 自动处理错误和重试

### 方案2：命令行批量下载
使用命令行脚本分批下载图片：

```bash
# 下载前100个图片
python scripts/download_images_batch.py 0 100

# 下载第101-200个图片
python scripts/download_images_batch.py 100 100

# 下载所有图片
python scripts/download_all_images.py
```

**参数说明**：
- 第一个参数：起始索引（从0开始）
- 第二个参数：下载数量

**优点**：
- 可以分批下载，避免一次下载太多
- 可以随时中断和继续
- 有详细的进度显示

### 方案3：单个图片下载
在产品编辑页面，点击"下载图片"按钮下载单个产品图片。

**优点**：
- 精确控制下载哪些图片
- 适合只需要部分图片的情况

## 注意事项

### 网络要求
- 需要稳定的网络连接
- 图片托管服务器: `cdn.ycmes.cn`
- 建议在非高峰时段下载

### 时间消耗
- 每个图片约0.5-1秒
- 1,667个图片预计需要10-20分钟
- 可以分批下载，避免一次占用太多时间

### 磁盘空间
- 每个图片约10-50KB
- 1,667个图片预计需要100-500MB
- 确保有足够的磁盘空间

### 错误处理
- 网络错误会自动跳过
- 失败的图片会记录日志
- 可以重新运行脚本继续下载

## 故障排除

### 问题1：数据库锁定
**症状**: `database is locked`
**解决**: 
- 关闭正在运行的应用
- 等待几秒后重试
- 或者使用分批下载脚本

### 问题2：网络超时
**症状**: `timeout` 错误
**解决**:
- 检查网络连接
- 增加超时时间
- 减少同时下载数量

### 问题3：磁盘空间不足
**症状**: `No space left on device`
**解决**:
- 清理磁盘空间
- 选择性下载重要图片
- 使用外部存储
