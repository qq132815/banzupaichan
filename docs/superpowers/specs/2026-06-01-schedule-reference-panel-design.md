# 排班参考面板设计方案

## 概述
在排班页面右侧将固定的"排班预警"面板改为5个Tab切换面板，为排班人员提供更多参考数据。

## 布局
保持现有"甘特图(2/3) + 右侧面板(1/3)"结构不变。

```
┌─────────────────────┬────────────────────────────────────┐
│                     │ [预警][其他班组][半成品][紧急][报工] │
│    甘特图 (2/3)     │────────────────────────────────────│
│                     │  Tab 内容区（可滚动）               │
├─────────────────────┴────────────────────────────────────┤
│                 📋 排班明细表格                            │
└──────────────────────────────────────────────────────────┘
```

## Tab定义

### Tab 1 — 排班预警
- 保持现有逻辑不变（生产需求 + 工单剩余量）

### Tab 2 — 其他班组排班
- 顶部日期选择器（默认今天）
- 按班组分组显示排班列表
- 不显示当前用户自己的班组
- 字段：设备、产品、工序、时间段、人员
- 数据源：现有 `/api/schedule` 接口，遍历其他 team_id

### Tab 3 — 近2天半成品产出
- 从工艺路线(process_routes)中取每个产品的工序列表(process_list)
- 取倒数第二道工序（仅一道工序则取该道）作为半成品工序
- 从报工(work_reports)中筛选最近2天内匹配该工序的记录
- 按班组分组显示
- 字段：产品名称、工序、报工数量、良品数、报工时间、操作人
- 新增API: `GET /api/reference/semi-finished-output`

### Tab 4 — 工单紧急度
- 未完成工单按交期升序（最紧急在上）
- 已超期标红
- 字段：工单号、产品、计划数/完成数、交期、优先级
- 数据源：现有 `/api/orders` 接口

### Tab 5 — 当日已完成报工
- 今天的报工数据，按时间倒序
- 字段：工单号、产品、工序、报工数、良品数、操作人、时间
- 数据源：现有 `/api/work-reports` 接口，带日期过滤

## 权限
所有角色（班组、计划员、管理员）都能查看所有Tab。参考数据不涉及操作权限。

## API设计

### 新增接口

#### GET /api/reference/semi-finished-output
返回近2天半成品报工数据。

逻辑：
1. 查询 process_routes，解析 process_list（逗号分隔）
2. 取每条路线的倒数第二道工序（仅一道则取该道）
3. 从 work_reports 中查询最近2天内匹配这些工序的记录
4. 按班组分组返回

响应格式：
```json
[
  {
    "team": "前段班组",
    "records": [
      {
        "product_name": "xxx",
        "process_name": "xxx",
        "report_qty": 100,
        "good_qty": 98,
        "create_time": "2026-06-01 10:30:00",
        "operator": "张三"
      }
    ]
  }
]
```

### 复用接口
- Tab 1: `/api/schedule-warnings` (已有)
- Tab 2: `/api/schedule?team_id=X&date=Y` (已有)
- Tab 4: `/api/orders` (已有)
- Tab 5: `/api/work-reports` (已有，带日期过滤)

## 前端实现
- schedule.html 右侧面板改为 Tab 结构
- 每个Tab有独立的内容区域和加载函数
- 切换Tab时按需加载数据（非全部预加载）
- Tab 2 日期变化时重新加载
- Tab样式与现有UI风格一致（pill按钮或underline样式）

## 文件变更
- `templates/schedule.html` — 右侧面板改为Tab结构 + JS逻辑
- `app.py` — 新增 `/api/reference/semi-finished-output` API
